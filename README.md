# Budget App — Personal Finance Dashboard

A self-hosted personal finance dashboard that parses your bank and credit card statements, tracks spending against budgets, and projects savings toward a financial goal.

---

## What It Does

| Tab | What You See |
|---|---|
| **Dashboard** | Income, expenses, contributions, net surplus, spending breakdown |
| **Monthly** | Month-by-month cash flow, click any month to see every transaction |
| **Budget** | Actual spend vs monthly limits per category |
| **Investments** | Portfolio holdings, cost basis, live or projected value |
| **Plan** | Savings trajectory toward a purchase or savings goal |
| **Settings** | Configure everything — income, plan, categories, statement folders |

---

## First-Time Setup (for developers / self-hosting)

### 1. Install Python
Download from [python.org](https://python.org) if you don't have it.

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Start the app
```bash
# Windows
python app.py

# Mac / Linux
python3 app.py
```
Then open your browser to `http://localhost:5100`

> **Note:** If you're using the PyInstaller-built `Budget.exe`, just double-click it — no Python needed. Right-click → Create Shortcut to add it to your desktop, then right-click the shortcut → Properties → Change Icon to use `Budget.ico`.

### 4. Configure Settings
Go to the **Settings** tab and work through the steps in order:

1. **Names** — Set Person 1 and Person 2 names
2. **Income** — Enter salary, tax rate, 401k % for both people
3. **Plan** — Choose plan type, set goal date and property details
4. **Year Incomes** — Set net monthly take-home per year for the plan projection
5. **Statements** — Drop your statement files in the folders below, then click ↻ Parse Statements

## Statement Folders

All folders live inside `BudgetApp/statements/`. Drop files in and click **↻ Parse Statements** in the top bar.

```
statements/
├── Credit/         ← Any credit card (Apple, Chase, Citi, Amex, etc.)
│                     Accepts: .pdf  .csv
├── Debit/          ← Any checking/debit account (Fidelity, Chase, BofA, etc.)
│                     Accepts: .csv  .pdf
└── Investments/    ← Fidelity Portfolio Positions snapshot
                      Accepts: Portfolio_Positions_MMM-DD-YYYY.csv
```

### How to Export Statements

**Credit cards:**
- Log in to your bank → Account Activity → Download / Export → CSV or PDF
- Apple Card: Wallet app → tap the card → export icon (top right)

**Fidelity Checking:**
- fidelity.com → Accounts & Trade → Account → Activity & Orders → Download → CSV
- Set date range to cover the full period you want to track

**Fidelity Portfolio Positions:**
- fidelity.com → Accounts & Trade → Portfolio → Positions → Download CSV
- Keep the default filename (`Portfolio_Positions_MMM-DD-YYYY.csv`) — the date is used to track snapshot age

## How the Parser Works

The `statements_parser` is a universal bank-agnostic parser. It:

1. Sniffs column headers (handles 30+ aliases for Date, Payee, Amount)
2. Auto-detects date formats (MM/DD/YYYY, YYYY-MM-DD, Jan 15 2024, etc.)
3. Handles split debit/credit columns and normalizes sign conventions
4. Falls back from table extraction to line-level regex for unusual PDFs
5. Scores each file with a confidence rating (HIGH / MEDIUM / LOW)

The `investments_parser` handles Fidelity's specific Portfolio Positions CSV format.

---

## Category Mapping

The `src/mapping.py` file contains 200+ merchant-to-category rules covering groceries, restaurants, utilities, transit, subscriptions, insurance, and more. It uses a two-pass system:

1. **High-priority rules** — applied first, unconditionally (e.g. your paycheck, specific excludes)
2. **Base mapping** — fills uncategorized transactions by payee name substring match

You can add custom categories and keyword rules in the **Settings → Custom Categories** card without editing any code.

## Plan Tab

The Plan tab projects your savings trajectory toward a financial goal. Three plan types:

| Type | Description |
|---|---|
| **Sell → Buy New Home** | Models current property appreciation + equity + savings toward a new home |
| **Save → Buy First Home** | Savings only — no property equity factored in |
| **Savings Goal Only** | General savings goal — no real estate modeling |

Set per-year income in **Settings → Income by Year** for a quick bulk update, then fine-tune individual months directly in the Plan tab table.

## GitHub Actions — Automated Builds

Every push to `main` automatically builds both platforms. Downloads appear under **Actions → Artifacts**.

### Setup Steps

**1. Create a private GitHub repository**
- github.com → New repository → Private
- Don't initialize with README

**2. Push your project**
```bash
cd BudgetApp
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/BudgetApp.git
git push -u origin main
```

**3. That's it**
The `.github/workflows/build.yml` is already in your repo. GitHub Actions picks it up automatically on your first push and builds both apps.

**4. Download the builds**
- GitHub repo → Actions tab → latest run → Artifacts section
- Download `Budget-Windows` or `Budget-Mac`
- Unzip and share the folder

### What Gets Distributed

The ZIP contains a clean app with no personal data:
```
BudgetApp/
├── Budget.exe  (or Budget.app on Mac)
├── settings.json        ← clean defaults, all zeros
├── savings_plan.json    ← clean defaults
├── overrides.json       ← empty
└── statements/
    ├── Credit/
    ├── Debit/
    └── Investments/
```

### Updating the App

```bash
git add .
git commit -m "describe your change"
git push
```

GitHub rebuilds both apps automatically (~5 minutes). Download new artifacts and share.

## Mac Note — Gatekeeper Warning

Because the app isn't signed with an Apple Developer certificate, Mac will show an "unidentified developer" warning on first launch. Users should:

1. Right-click `Budget.app`
2. Click **Open**
3. Click **Open Anyway**

This is only needed once. After that it opens normally.

## Privacy

All data stays on your machine. No data is sent anywhere except:
- Stock prices (fetched from Yahoo Finance via `yfinance` when you click "Fetch Live Prices")
- Nothing else — no analytics, no telemetry, no cloud sync
